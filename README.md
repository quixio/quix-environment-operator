# Quix Environment Operator

<!-- solution-docs:begin readme -->
A Kubernetes controller that lets the Quix platform request isolated customer-hosted environments without receiving broad cluster access. Each environment request becomes a namespace plus scoped access for the platform identity, bounded by permissions the customer can review before installation.

**Note:** This project is open source and can be used by anyone, but it is primarily designed for use within the Quix ecosystem.

This exists because handing the platform cluster-admin, or asking customers to create every namespace and binding manually, makes the security boundary either too broad or too operationally expensive. The operator’s differentiator is that the customer approves the permission model once, while the platform can still provision environments on demand inside that boundary.

## Key features

- [Environment provisioning](docs/features/environment-provisioning.md) — declarative namespace creation with strict naming, ownership tracking, and controlled metadata propagation
- [Scoped platform RBAC](docs/features/scoped-platform-rbac.md) — per-namespace RoleBindings to a pre-approved ClusterRole, with privilege-escalation validation
- [Lifecycle & status](docs/features/lifecycle-and-status.md) — phase tracking, Kubernetes events, and finalizer-driven cleanup
